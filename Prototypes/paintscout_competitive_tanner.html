Closing the Gaps

Features PaintScout has today — DripJobs response

Tax Rate Entry

Inline per-proposal modal with template inheritance & source toggle. Tax flows to Customer Portal, PDF, and invoices.

86b7u2fr9
Triage
Summary
Replace the Company Settings-level tax rate system with a simplified modal-based tax entry experience. The Tax Rate row in the Payment Settings card displays the current saved rate alongside an Edit button. Clicking Edit opens a modal where the user enters a percentage, validates it, and saves. Totals update only on Save. Proposal Templates support the same pattern. There is no Tax Name field — the tax line in Totals always displays as "Tax."
User Story
As a user viewing the Proposal Builder, I want to click Edit next to the Tax Rate setting and enter a percentage in a modal, so that I can set tax for this job without accidentally changing it while browsing settings.

As a user creating or editing a Proposal Template, I want to set a Tax Rate via the same Edit modal experience, so that I can pre-configure a reusable template for a specific tax rate.

Named Discount Library

Company-level discount library selectable in Apply Discount modal. Replaces ad-hoc entry with consistent, labeled discounts.

86ba231jg
Triage
Summary
Allow contractors to define a reusable library of named discount types in Company Settings. Each entry includes a label and a percentage rate. When applying a discount, users select from their saved library rather than entering a rate manually. Manual ad-hoc entry remains available alongside library selection. Library discounts carry their label to the Customer Portal and PDF.
User Story
As a contractor configuring my account, I want to define a list of named discounts (e.g., "Scheduling Incentive — 10%", "Veteran Appreciation — 10%"), so that my team can apply consistent, correctly-labeled discounts to every proposal without memorizing or mistyping rates.

As a user applying a discount in the Proposal Builder, I want to select a discount from my saved library, so that the name and rate are applied automatically with the correct label visible to the customer.

Proposal Expiry Enforcement

Block customer signing on expired proposals. Re-Open flow mirrors rejected proposal pattern. UI placement already solved in new Proposal UI.

86b2xn7hx
Triage
Summary
The expiration date field exists and is surfaced in the new Proposal UI header row. Currently it is informational only. This ticket makes enforcement real: expired proposals block customer signing in the Customer Portal, restrict contractor edits and resends, and surface a clear expired state with a Re-Open flow (set new date → edit → re-send) mirroring the existing rejected proposal pattern.
User Story
As a contractor who set an expiration date, I want the system to enforce that deadline automatically, so that pricing is protected and customers understand the offer is no longer valid without my re-approval.

As a customer viewing an expired proposal, I want to see a clear message explaining the proposal has expired, so that I know to contact the contractor rather than attempting to sign.

Round Gallons + Overage Monetization

Shows Rd. Gallons, Overage (gallons delta + dollar cost), and row-selection in the Product Report. Contractors select rows and add overage costs directly to the proposal as individual or combined touch-up line items. Locks at data layer for future Change Orders.

86azcxv4c  ·  Shipped May 11, 2026
Shipped
Summary
A "Show Round Gallons" toggle in the Product Report reveals three unified columns: Rd. Gallons, Overage (gallons delta + dollar cost stacked), and a row-selection checkbox. Selecting rows surfaces a floating action panel with two options: Add as Individual Line Items or Add as Single Touch-Up Line Item. Committed rows lock with a ✓ Added badge. Packages mode disables the toggle. Once overage is added, rounding is locked at the data layer for any future Change Orders.
User Story
As a painting contractor, I want the Product Report in my proposal to show how many whole gallons of each product I actually need to purchase, and the cost of that overage, so that I can accurately order materials and optionally recover those costs in the proposal without manual calculation.

DripJobs Advantages

What PaintScout doesn't have

Discount Line Item Expiry + Countdown Timer

Per-discount expiry dates with a live countdown in the Customer Portal. Proposal stays live — only the incentive expires. Proposal total recalculates automatically.

★ Beyond PaintScout
86ba25xnm
Summary
Allow contractors to set an optional expiration date on individual discount line items. When a discount expires, it grays out in the Customer Portal with a visible countdown timer (days, hours, minutes) while the proposal itself remains live and signable. The proposal total recalculates to remove the expired discount. The contractor receives an in-app notification when a discount on a sent proposal lapses. Multiple discounts each expire independently. Gated on 86b2xn7hx shipping first.
User Story
As a contractor applying a discount, I want to set an expiration date on that discount so that customers feel urgency to sign before the incentive disappears, without making the whole proposal inaccessible.

As a customer viewing a proposal, I want to see how long a discount is valid so that I understand the cost of waiting to approve.

Proposal Timeline Fields

"If customer approves by" + Proposed Start/End Dates in the Customer Portal. Structured urgency vs. a hard cutoff. Auto-calculates start date from approval deadline.

★ Beyond PaintScout
86b15mryv

Setting the approval deadline auto-calculates the start date as 7 days later. Pipeline & Automation — Included on All Plans

40+ pre-built drips, automated deal pipeline, DJ Chat, SMS. PaintScout CRM is +$42–49/mo with no native drip automation whatsoever.

Why This Matters
PaintScout is estimating-first. Its CRM is a paid add-on (+$42–49/mo) with no built-in drip sequences, no automated follow-up, and no SMS. A PaintScout contractor on Day 1 has zero automated follow-up. A DripJobs contractor on Day 1 has 40+ pre-built drips running across their entire pipeline without any configuration. DripJobs' automated pipeline is included on all plans — Pro, Growth, and Scale — at no extra charge. DJ Chat, SMS blasts, and Blast campaigns are also native, with no third-party messaging tool required.
✦ Strategic moat — no ticket needed

Job Costing, ACH, Deposits & Change Orders

Full financial operations built in. PaintScout has no job costing. DripJobs $97/mo vs PaintScout $141–168/mo for a comparable feature set.

Why This Matters
PaintScout has no job costing module. DripJobs includes job costing, ACH payments, deposits, and change orders on applicable plans. At comparable feature levels: DripJobs costs ~$97/mo vs PaintScout's ~$141–168/mo when adding the CRM add-on. Change Orders are a particularly meaningful differentiator — contractors can issue scope changes, get customer signature, and track them against the original proposal, all within the same platform. PaintScout requires leaving the estimating tool entirely to handle any post-estimate financial workflow.
✦ Strategic moat — no ticket needed

Overage Monetization — Revenue Recovery Built In

PaintScout rounds products on the work order only. DripJobs lets contractors select per-product overage costs and add them to the proposal as line items in one action.

★ Beyond PaintScout
Why This Matters
PaintScout's rounding feature applies to the work order only and is a global calculation preference — it rounds product quantities to avoid compound errors. It has no mechanism for the contractor to recover the cost of that rounding. DripJobs ships a complete flow: the contractor sees the per-product overage cost in the Product Report, selects which rows to charge for, and adds them to the proposal as "Sherwin Williams ProMar 400 Interior (touch-up)" line items or a single combined "Leave behind touch-up product" line item — all in two clicks. The feature is tied to ticket 86azcxv4c, shipped May 11, 2026.
